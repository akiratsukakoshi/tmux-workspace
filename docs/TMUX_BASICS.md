# tmux 基本ガイド（初心者向け）

このリポを使う上で「最低限ここまで分かっていれば OK」というレベルで tmux の仕組みと操作をまとめた入門。
詳しくは `man tmux` や公式 wiki を参照。

---

## 1. tmux とは何か（一言で）

**「1つのターミナル画面の中で、複数の画面を分割しつつ、ターミナルを閉じても中身が生き続ける」ためのツール。**

特に WSL では「PC スリープしてもセッションが生きてる」「VSCode を閉じても tmux の中の処理は走り続けている」が便利。

---

## 2. レイヤー構造

```
[1] tmux server               ← バックグラウンドに常駐するプロセス。1つの WSL に基本1つ
 └ [2] session                ← 名前付きの作業セット。複数同時に存在できる
    └ [3] window              ← ブラウザのタブみたいなもの。1セッション内に複数可能
       └ [4] pane             ← ウィンドウの分割ブロック。ここでシェルが動く
```

このリポでは：
- セッション = `claude-parallel`（1つ）
- ウィンドウ = 1つ
- ペイン = 4つ（4プロジェクトに対応）

---

## 3. 起動と attach の違い

```
[A] tmux サーバ + セッションを「作る」     ← 初回、or 新セッション
[B] 既存セッションに「繋ぐ」(attach)        ← 2回目以降、再接続
```

サーバが居なければ `tmux` コマンドが自動で立ち上げます。サーバを意識的に起動する必要はありません。

### サーバが死ぬタイミング

| 何をした | サーバは |
|---|---|
| `tmux` から `prefix d` で detach | **生きてる** |
| ターミナルウィンドウを閉じる | **生きてる**（WSL 自体は裏で残る） |
| Windows をサインアウト | だいたい生きてる |
| **PC をシャットダウン／再起動** | **死ぬ** |
| `wsl --shutdown` を実行 | **死ぬ** |
| WSL のアイドル自動停止 | **死ぬ** |
| `tmux kill-server` | 死ぬ |

つまり「PC を再起動したら tmux も死ぬ」が原則。各エージェントの会話履歴自体はディスクに残っているので、再起動後は `start.sh` で立ち上げ直して、各エージェントで `--continue` 系コマンドで前回の続きから再開すれば OK。

## 4. prefix キー

tmux のキー操作は全部「**prefix を押してから、次のキーを押す**」という2段階。
デフォルトの prefix は **`Ctrl-b`**（このリポでも変更していません）。

表記の慣習：
- `prefix d` = `Ctrl-b` を押して離してから `d` を押す
- `prefix Ctrl-d` = `Ctrl-b` の後に `Ctrl-d`

なぜ prefix がある？ — tmux 内で動く普通のシェルも `Ctrl-` キーを使うので、tmux 宛のキーかシェル宛かを区別する前置きが必要。

---

## 5. 最低限のキー一覧

このリポを使うだけならこれだけ覚えれば動かせる：

### tmux の中で使うキー

| キー | やること |
|---|---|
| `prefix d` | detach（画面から離れる、セッションは生きる） |
| `prefix 矢印` | 隣のペインに移動 |
| `prefix z` | アクティブペインを最大化／戻す（ズーム） |
| `prefix =` | レイアウトを 2x2 に再均等化（このリポで追加） |
| `prefix Alt-1〜5` | tmux 標準のプリセットレイアウト切替 |
| `prefix r` | tmux.conf を再読み込み（このリポで追加） |
| `prefix ?` | 全コマンド一覧 |

マウスも有効化済み。クリックでペイン切替、境界ドラッグでサイズ変更可能。

### シェルから使うコマンド（tmux の外）

```bash
tmux ls                              # セッション一覧
tmux attach -t <name>                # 名前指定で attach
tmux a                               # 省略形（1個しかなければそれに繋ぐ）
tmux new -s <name>                   # 新規セッション作成 + attach
tmux kill-session -t <name>          # 特定セッションを殺す
tmux kill-server                     # 全部終了
```

## 6. 5分の練習

このリポと無関係に、tmux 単体で挙動を掴むのに使える練習：

```bash
# 1. 空セッションを作る
tmux new -s practice

# 2. 中に入ったら以下を試す:
#    prefix "             ← ペインを水平分割（上下に2つ）
#    prefix %             ← ペインを垂直分割（左右に2つ）
#    prefix 矢印キー       ← ペイン間移動
#    prefix z             ← ズーム / 解除
#    prefix d             ← detach (シェルに戻る)

# 3. detach 後の確認:
tmux ls                              # practice が居る
tmux attach -t practice              # また入れる

# 4. 後片付け:
tmux kill-session -t practice
tmux ls                              # 'no server running' になる
```

これでだいたい掴めます。

## 7. このリポの設定で変えていること

参考までに、このリポの `tmux.conf` で標準と違う設定：

- `Ctrl-b` の遅延を `escape-time 10` で短縮
- `mouse on` でマウス操作有効化
- `base-index 0` / `pane-base-index 0` でウィンドウ／ペイン番号を0から
- `status-interval 1` でステータスバーを毎秒更新
- `pane-border-status top` で各ペイン上に枠＋タイトル表示
- `prefix r` で tmux.conf 再読み込み
- `prefix =` で tiled レイアウト即適用

詳細はリポルートの [`tmux.conf`](../tmux.conf) を読んでください。
